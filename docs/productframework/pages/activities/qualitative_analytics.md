## Description

Качественная аналитика продукта предназначена для понимания эффективности процессов получения ценности от продукта, позволяя определить причины поведения потребителей и их мотивацию при принятии решений.

## Extracted Text

QUALITATIVE ANALYTICS

Почему именно так потребители взаимодействуют с продуктом, а не как-то иначе?

Артефакты и документы

Идеи по продукту от конечных потребителей или сотрудников компании. Как правило, категоризируются для дальнейшей обработки и/или передачи в другие отделы. Например, конвертация в баги или беклог продукта.

Обращения из поддержки

Обратная связь, сообщения об ошибках, уведомления об эффективности процесса продаж, информация о мотивации клиентов и т.д., которая приходит через любые каналы сопровождения клиентов.

Модели и инструменты

Метод сортировки обраной связи по эмоциональной оценке (положительная / отрицательная) и необходимости исправления. Обратная связь распределяется по четырем категориям: совет, комплимент, критика и предложение. Советы - положительная оценка, но требует улучшения. Комплименты - сильное качество продукта, ничего не нужно делать. Критика - отрицательная оценка, требуют обязательного исправления. Предложение - отрицательная оценка, но с этим можно жить.

Метод поиска изначальных причин проблемы, заключающийся в последовательном задавании вопроса "Почему это произошло?" для каждого из ответов пять раз подряд. Уровней может быть больше или меньше, чем пять, пока не будет найдено причина поведения, поддающееся изменению. Каждый из ответов должен быть фактом, а не гипотезой. На каждый из вопросов может быть приведено несколько ответов, тем самым формируя не цепочку причинно-следственных связей, а дерево.

Формат описания потребностей клиентов, в котором основной акцент стоит на контексте, при котором возникают эти потребности. Формат был предложен как альтернатива user story и смещает фокус с особенностей портрета клиента на контекст работы, которую ему нужно выполнить, и почему эта работа для него важна.
